# Product Guidelines: ComicInfoXmlGenerator\n\n## Prose & Voice\n- **Tone**: Professional and technical.\n- **Communication Style**: Clear, direct, and unambiguous. Use technical terminology when it helps precision (e.g., 'XML Schema Validation failed' instead of 'The file is broken').\n- **Status Messages**: Informative and actionable. Every message should clearly state what happened and its result.\n\n## Visual Design & UI\n- **Principles**: Simple and familiar. The interface should feel intuitive by following established desktop software conventions (e.g., standard Material Design).\n- **Aesthetics**: Focus on clarity and readability. Prioritize metadata tables and list views that are easy to scan.\n- **Icons & Color**: Use standard symbolic meanings (e.g., green for success, red for errors, amber for warnings) to provide instant visual feedback.\n\n## User Experience (UX)\n- **Workflow First**: Optimize for high-volume tasks. Minimize necessary clicks for common operations like 'Scan' and 'Commit All'.\n- **Explicit Consent**: Never modify an archive file without explicit user confirmation. Always show a summary of changes before writing to disk.\n- **Consistency**: Ensure all parts of the application use the same metadata field names and behaviors.\n\n## System Behavior & Error Handling\n- **Robust Logging**: Maintain verbose logs of all operations. In case of a failure during bulk processing, provide enough detail for the user to troubleshoot the specific file or scraper that caused the issue.\n- **Resilience**: The tool should continue processing other files even if one file fails, unless the error is a systemic failure (e.g., no internet for scrapers).\n- **Transparency**: Always display the source of any metadata (e.g., 'Filename Scraper', 'ComicVine API') so the user can trust or verify the information.
